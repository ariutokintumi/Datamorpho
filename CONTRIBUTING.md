# Contributing

**Website:** https://datamorpho.io  
**Repository:** https://github.com/ariutokintumi/datamorpho  
**Discussions:** https://github.com/ariutokintumi/datamorpho/discussions  
**Issues:** https://github.com/ariutokintumi/datamorpho/issues

Datamorpho is an open protocol project. Contributions are welcome in any of the following areas.

## What can be contributed

- **Specification clarity** — wording improvements, ambiguity fixes, edge-case coverage
- **Whitepaper improvements** — factual corrections, structural suggestions, coverage gaps
- **Reference implementation** — Python tooling bugs, test coverage, correctness issues
- **Examples and vectors** — new carrier examples, interoperability vectors, test fixtures
- **Documentation** — FAQ additions, glossary corrections, README improvements
- **Website** — copy corrections, accessibility fixes

## How to contribute

1. Open a [GitHub Issue](https://github.com/ariutokintumi/datamorpho/issues) first for anything non-trivial to discuss the change before implementing it.
2. For small fixes (typos, wording), a direct pull request is fine.
3. For protocol-level changes, use [GitHub Discussions](https://github.com/ariutokintumi/datamorpho/discussions) so the reasoning can be reviewed before any PR.

## Contribution principles

- Correctness before polish.
- Protocol clarity before feature breadth.
- Open documentation before closed assumptions.
- All contributions are subject to review and may be declined if they conflict with the protocol design or current implementation constraints.

## Current priorities

1. Specification correctness and coherence
2. Python reference implementation stability and test coverage
3. Examples and interoperability vectors
4. Browser-compatible JavaScript library (derived from the tested Python implementation)

## Contact

For questions before contributing, reach out via [GitHub Discussions](https://github.com/ariutokintumi/datamorpho/discussions) or at g@evvm.org.
